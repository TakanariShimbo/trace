# Sangaku（山岳） — 日本の山を見て知るアプリ

国土地理院の地形データ（**DEM＋航空写真**）を使い、日本の山を「見て・知る」ためのアプリ。
3D地形のシームレスなズーム閲覧を土台に、太陽・月の動き、写真やライブカメラへ山名を重ねる
AR、オフライン保存までを1つにまとめている。用途別のホームから各モードへ分岐する:

- **地形を見る**: 日本全体〜街レベルまでシームレスにズーム詳細化する3D地形。立って一人称でも見回せる。
- **太陽・月の動きを見る**: 任意の地点・日時の太陽/月の方位・高度、日の出・日の入り。
- **写真に山名をのせる / カメラで山名を見る**: 撮影写真やカメラ映像へ、見えている山の名前を重ねる（AR）。
- **オフライン保存**: 見たい範囲を事前ダウンロードし、通信なしで3D表示。

描画の中核は **XYZタイルのクアッドツリーLOD**。タイルごとにDEMメッシュ＋航空写真を持ち、
カメラ距離（画面上の見かけサイズ）に応じて4分割/統合し、非同期にストリーミングする。
対象は日本（ルートタイルのbbox）に限定。

## 動作の仕組み

平面 Web メルカトル（EPSG:3857）にタイルを並べ、原点（日本中心）相対の「km相当」座標で3D化する。

1. **ルート**: 日本bboxに重なる z5 タイル群（十数枚）をクアッドツリーの根とする。
2. **LOD選択（毎フレーム）**: 各タイルの画面上の大きさ(px)を見積もり、`SPLIT_PX` を超えれば
   4つの子タイル（z+1）へ分割。視錐台外のタイルは描画しない（少し外側まで先読み）。
3. **ストリーミング**: 子のDEM/航空写真が揃うまで親を表示し、揃ったら一斉に差し替える
   （常に隙間なく描画＝プログレッシブ詳細化）。スカートでLOD境界のひび割れを隠す。
4. **メモリ**: 一定フレーム未使用のタイルは mesh/texture を破棄。

DEMは z14 がネイティブ（約10m）。航空写真は z16 まで使うので、拡大すると地形より先に
**画像が高精細化**していく（Google Earth と同様）。

## データ出典（国土地理院・APIキー不要・CORS開放）

- 標高(DEM): `xyz/dem`（`.txt`、z0〜14）
- ベースマップ: `xyz/seamlessphoto`（航空写真）/ `xyz/std`（標準地図）/ `xyz/pale`（淡色）/ `xyz/hillshademap`（陰影起伏）
- 地名検索: `msearch.gsi.go.jp/address-search/AddressSearch`

> 出典表記: 「地図・航空写真・標高データ：国土地理院タイル」。

### 山の解説（写真ARのキャプション）

- `public/data/mountain_descriptions.json` — 各山の解説文（Wikipedia 日本語版の冒頭抜粋、921山）。
  写真AR（仕上げ）で山名・標高とともに下部キャプションに表示し、写真にも焼き込む。
- 取得は `scripts/fetch-descriptions.mjs`（`node scripts/fetch-descriptions.mjs`）。山名＋座標で記事を照合する。
- テキストは Wikipedia 由来のため **CC BY-SA 4.0**。各エントリに出典記事 `url` を保持し、UI/焼き込み画像に「出典: Wikipedia（CC BY-SA）」を明記する。

## 主な構成

- `src/lib/mercator.ts` — メルカトル/経緯度/タイル/ワールド座標の変換（DOM非依存・ワーカー共有）
- `src/lib/demTiles.ts` — DEMタイル取得＋Cache APIキャッシュ
- `src/lib/basemaps.ts` — ベースマップ各レイヤの定義・タイル取得・Cache APIキャッシュ
- `src/lib/tileSampler.ts` — 1タイルの標高グリッド生成（ワーカー実行）
- `src/workers/terrain.worker.ts` / `terrainClient.ts` — 地形サンプリングのワーカープール
- `src/terrain/QuadtreeTerrain.ts` — クアッドツリーLODエンジン（中核）
- `src/components/MapView.tsx` — Three.js セットアップ＋地図的カメラ操作（MapControls）

## 操作（UI）

- ホーム画面の5枚のカードから用途別モードへ。各モードは共通の3Dエンジン上で振る舞いを切り替える。
- ホーム画面下部の **表示設定** パネルで、中心マーカー・山頂マーカー・空のグラデーション・標高の誇張（地図／風景）を変更でき、各モードに引き継がれる（設定はブラウザに保存）。
- 各モードの画面上部は **ホーム** ボタンのみ。操作は下部の**ドックパネル**に集約し、
  パネルは**ドラッグで移動・折りたたみ**できる（1モード1パネル）。
- ドック内は折りたたみ帯（セクション）のリスト: **案内 / 操作（地図・風景／3D・2D／現在地ほか）/
  検索 / 地図（ベースマップ）/ 太陽・月 / カメラ設定** など、モードに応じて並ぶ。
  - **検索**: 検索バー内で「山名／土地名／全て」を切替→候補タップでその地点へフライト。
    - 山名: 山岳データ(`public/data/mountains.json`、約1,061山)を名前・読みで部分一致（標高・都道府県付き・オフライン可）
    - 土地名: 国土地理院 地名検索API ／ 全て: 山→土地名の順でマージ
  - **ベースマップ**: 航空写真／標準地図（登山道・地名）／淡色地図／陰影起伏
- ドラッグ: 地図のパン・回転・傾き、ホイール／ピンチ: ズーム。「風景」に切り替えると、その場に立ったように見回せる。

## PWA / オフライン（事前ロード）

- **PWA**: `vite-plugin-pwa` でアプリ本体（シェル）をプリキャッシュ。インストール可能で、
  一度読み込めばオフラインでも起動する。
- **事前ロード**: ホームの「**オフライン保存**」モードから、**画面中央（見ている地点）を中心＋半径(km)**で一括ダウンロード。
  DEM・航空写真を Cache API（`gsi-dem-tiles-v1` / `gsi-aerial-tiles-v1`）に保存し、以降は
  通信なしでもその範囲を3D表示できる（描画パスは常にキャッシュ優先なので分岐不要）。
  - 対象範囲は地面に**円のインジケータ**で表示。半径・最大ズームはスライダーで調整。
  - タイル数・サイズ目安を事前表示。広すぎる場合は上限で打ち切り警告。
  - 保存済み容量の表示とキャッシュ削除も可能。
  - 注意: GSIのDEMタイルは陸域で約 **512KB/枚** と重い。広域×高ズームは数GBになるため、
    登山エリアにズームインしてから保存するのが現実的。
